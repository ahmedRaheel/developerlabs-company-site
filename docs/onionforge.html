OnionForge Enterprise AI

OnionForge is an enterprise .NET code generator for:

  • Onion Architecture
  • Domain-Driven Design
  • CQRS
  • MediatR
  • FluentValidation
  • EF Core
  • Repository + Unit of Work
  • Minimal APIs
  • API Controllers
  • Domain Events
  • Outbox Pattern
  • Serilog
  • Health Checks
  • OpenTelemetry-ready structure
  • Docker-ready structure
  • AI-assisted focused code generation

Build

bash
dotnet restore
dotnet build

CLI Commands

bash
onionforge new-solution --name EStore --namespace EStore --output "D:\Projects\EStore"

onionforge add-aggregate --name Product --module Catalog --properties "Name:string,Price:decimal"

onionforge add-command --name CreateProduct --module Catalog --properties "Name:string,Price:decimal"

onionforge add-query --name GetProductById --module Catalog --properties "Id:Guid"

onionforge add-handler --name CreateProduct --module Catalog --type command --properties "Name:string,Price:decimal"

onionforge add-endpoint --name CreateProduct --module Catalog --type command --http post --route "/api/catalog/products"

onionforge add-crud --entity Product --module Catalog --properties "Name:string,Price:decimal"

onionforge add-outbox

onionforge add-docker

onionforge add-otel

onionforge ai-code --module Catalog --name CreateProduct --kind command --prompt "Create product with name, sku and price validation"

AI Setup

cmd
setx OPENAI_API_KEY "your_api_key"
setx ONIONFORGE_AI_MODEL "gpt-4o-mini"

AI generation is intentionally focused. It writes exactly one C# file based on the requested kind.

Package CLI

bash
dotnet pack src/OnionForge.Cli -c Release -o ./nupkg
dotnet tool install --global --add-source ./nupkg OnionForge.Cli

Visual Studio VSIX

The VSIX shell calls the CLI. Build it from Visual Studio with Visual Studio extension development workload installed.
